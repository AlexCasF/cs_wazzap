Wazzap

Project summary for the Python socket chat application. Tested Kali IP: 192.168.106.128

What the App Does

Wazzap is a terminal chat app built with Python sockets. The server runs on the Kali VM and the client runs on the Windows host. Both sides can send and receive text messages in real time over LAN or through a Pinggy TCP tunnel.

How to Run It

  • Server on Kali: python3 server.py 0.0.0.0 9999
  • Client on Windows: python client.py 192.168.106.128 9999
  • Use /quit to close the chat cleanly.

Pinggy

Pinggy TCP tunneling worked at the beginning and allowed the client to connect to the Kali-hosted server over the internet. Later attempts were inconsistent because the current server handles one client session and then exits, so it must be restarted before reconnecting.

Features Added

  • Real-time two-way chat with Python sockets
  • Simultaneous send/receive using select
  • Configurable host and port arguments
  • /quit for clean shutdown
  • /leet to toggle leetspeak for future outgoing messages
  • /virus prank effect with warning screen and glitch animation

What I Learned and Challenges

  • I learned how client/server socket communication works in Python.
  • I learned how to use select so the app can send and receive without blocking.
  • I learned how to connect the Windows host to the Kali VM over LAN and how to use scp and Pinggy.
  • I fixed an early port mismatch between client and server.
  • I worked around the difference between Windows console input and Linux stdin behavior.
  • Pinggy testing was harder because the server currently accepts one client session and then exits.
